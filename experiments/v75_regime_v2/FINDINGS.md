# v7.5 Regime Detector v2 — Findings

**Date**: 2026-05-01
**Status**: ❌ Hypothesis disproven. No deployment.

## Problem we tried to solve

On 2026-04-30 / 2026-05-01 gold rallied yesterday (~+0.5%) then reverted hard
today (~-0.5%). Oracle XAU, Midas XAU and Janus XAU all bought into the
falling market because:
- Production K=5 K-means cluster classifier said "C4 HighVol"
- Option D's directional gate uses the trailing 288-bar (24h) return,
  which was still positive (yesterday's rally dominated the window)
- → R3* uptrend rules got armed → all three products went LONG

We hypothesised a "smarter regime detector" could anticipate these shifts.

## What we built and tested

| # | Approach | Folder | Result |
|---|---|---|---|
| 0 | 16-feature per-bar fingerprint matrix (multi-scale momentum + slope acceleration + vol ratio + position-in-range + HH/LL count + streak + trend consistency) | `00_compute_rich_fingerprints.py` | 586,488 bars × 18 cols computed |
| 1 | K-means K∈{5,6,7,8} on the rich features | `01_train_kmeans.py` | All trained, K=7 looked finest |
| 2 | Side-by-side classification vs production v1 + edge analysis | `02_compare_to_v1.py` | K=7 caught 56% of v1 wrong-side bars (vs ~37% baseline) **but** the resulting "BearBreakdown" cluster had no forward-1h directional edge (whipsaw, drop24% / rise27%) |
| 3 | Gaussian HMM with N∈{3,4,5} hidden states, transition-aware | `03_train_hmm.py` | Best state mean fwd-1h = +0.026% — too small to overcome spread/cost |
| 4 | Daily-timeframe veto (block longs on Down days, shorts on Up days, threshold sweep ±0.5% to ±3%) | `04_daily_veto.py` | At every threshold the veto removed *net profitable* trades. Stripped 107-344R from Oracle (worth -107R to -344R) over 15-month holdout |

## Quantitative summary

```
Oracle XAU 15-month holdout (1,367 trades, post-2024-12-12 unseen):
  Baseline:                     PF=3.48  WR=65.3%  R=+954  DD=-16.7
  + K-means K=7 regime:         no clean implementation; clusters not directional
  + HMM N=4 best state filter:  similar (no exploitable edge)
  + Daily veto ±0.5%:           PF=3.36  R=+610  → -344R worse
  + Daily veto ±1.0%:           PF=3.36  R=+641  → -314R worse
  + Daily veto ±2.0%:           PF=3.19  R=+719  → -235R worse
  + Daily veto ±3.0%:           PF=3.38  R=+848  → -107R worse

Midas XAU 27-month holdout (2,636 trades): same pattern — every veto setting
loses profit.
```

## Why every approach failed

At gold M5 timescales, returns are ~random-walk with very short momentum
(<10 bars) and reversion at longer scales. Any regime detector trained on
price/volatility statistics finds clusters that *describe* the recent past
but the resulting state has **no exploitable forward-1h directional edge**.

The Oracle/Midas confirm heads (per-cluster XGBoost on 18 v72L physics
features) already extract what's available at this timescale. Adding a
regime overlay on top doesn't help because there's no orthogonal signal
left.

## Conclusion

**There is no simple regime fix for the May 1 type of loss day.**

What looked like a regime-detection failure is actually normal model
variance. The validated holdout drawdown is -16.7R; the May 1 14-hour
loss is smaller than that and well within the trained distribution. The
trained models already do the best regime conditioning possible from M5
data; layering a slower regime classifier on top removes more profit
than it saves.

## What this implies for production

1. **Don't deploy a regime overlay.** Empirically harmful at every
   threshold/architecture we tried.
2. **Accept the variance.** Bad days are part of the validated PF 3.48 /
   PF 2.24. Track 30+ day rolling P&L, not 14-hour windows.
3. **v2 multi-slot is correct.** It captures more sample size per period
   without changing per-trade expectancy. Long-run results should track
   the holdout closely.
4. **The next worthwhile experiment** would be **cross-instrument signal**
   (DXY / SPX / US10Y as inputs to a *separate* regime gate). That's a
   different feature space, not the same M5 data we already use. If
   anyone wants to tackle it later, that's the direction with most
   potential remaining edge.
